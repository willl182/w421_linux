# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Durante la configuracion aparecieron fallas recurrentes por rutas incorrectas de config, confusion de credenciales (password vs passphrase vs token), sintaxis antigua de `tailscale serve` y permisos Docker insuficientes para `adminops`.

## User-Requested Corrections

- Documentar solo el plan final que si funciono.
- Proponer paginas de Logseq necesarias para operar el stack.
- Capturar aprendizajes y errores para no repetirlos.
- Dejar todo versionado en git con flujo local -> remoto -> push.

## The Solution / Workaround

1. Confirmar rutas reales con `docker inspect` antes de editar archivos.
2. Usar clave SSH para login operativo y separar tipos de credencial.
3. Usar sintaxis actual: `tailscale serve --bg http://127.0.0.1:57086`.
4. Ejecutar Docker con `sudo` o agregar `adminops` al grupo `docker`.
5. Verificar salud con comandos canonicos en vez de confiar solo en paneles externos.

## Learnings to Preserve

- "Offline" en portal no siempre significa servicio caido.
- Capa red y capa app deben endurecerse juntas (defensa en profundidad).
- El orden local -> commit -> remoto -> push evita confusiones al aprender git.
