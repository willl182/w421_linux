# Technical Finding / Milestone

**Context**: Recuperacion de OpenClaw Web UI + Telegram en VPS

## The Discovery

El estado "offline / disconnected from gateway" no estaba relacionado con puertos o Tailscale, sino con autenticacion/pairing WebSocket (`code=1008 reason=pairing required`).

Tambien se confirmo que los errores Telegram `getMe 401/404` se deben a token invalido o token no recargado en runtime.

Flujo estable validado:

1. Confirmar red/app (`docker ps`, `curl 127.0.0.1:57086`, `tailscale serve status`).
2. Resolver pairing en canal correcto (`--channel telegram`) dentro del contenedor.
3. Si cambia `.env`, recrear contenedor (`up -d --force-recreate`) y verificar con `getMe`.

Resultado final: `getMe` en `ok: true` para bot activo `@w421claw_bot`.
