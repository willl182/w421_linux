# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Durante la recuperacion de acceso aparecieron sintomas mezclados:

- UI en "offline/disconnected from gateway".
- Logs con `pairing required`.
- Telegram en loop con `getMe failed (401/404)`.

Esto genero confusion entre red/puertos, auth, y token de Telegram.

## The Solution / Workaround

1. Separar diagnostico por capas: red primero, auth despues, canales al final.
2. Aceptar que `pairing required` es auth/session de gateway, no firewall.
3. Validar Telegram contra API directa (`getMe`) para evitar suposiciones.
4. Re-crear contenedor (`up -d --force-recreate`) cuando cambios de `.env` no parecen reflejarse.

## Learnings to Preserve

- Evitar pegar placeholders en `.env` durante incidentes.
- No mezclar comandos del sistema con texto no ejecutable.
- Guardar un runbook corto con comandos validados para reducir friccion en futuras recuperaciones.
