# Technical Finding / Milestone

**Context**: Cierre formal de sesion con hardening SSH/fail2ban ya operativo

## The Discovery

El hardening real no queda cerrado solo por editar archivos; debe verificarse en runtime con comandos de estado efectivos.

Secuencia que confirma estado sano:

1. `sudo sshd -T | grep -E '^(passwordauthentication|permitrootlogin|pubkeyauthentication|kbdinteractiveauthentication|port) '`
2. `sudo systemctl is-active fail2ban`
3. `sudo fail2ban-client ping`
4. `sudo fail2ban-client status sshd`
