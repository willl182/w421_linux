# Technical Finding / Milestone

**Context**: Hardening SSH/UFW en VPS Hostinger

## The Discovery

Aunque se agrego `99-hardening.conf`, el valor efectivo seguia en `passwordauthentication yes`.

La causa fue una configuracion previa de cloud-init:

- `/etc/ssh/sshd_config.d/50-cloud-init.conf: PasswordAuthentication yes`

En este host, el comportamiento observado fue que la primera ocurrencia aplicada mantuvo el valor en `yes`.

## The Solution

1. Deshabilitar archivo conflictivo de cloud-init (`50-cloud-init.conf`).
2. Validar config con `sshd -t` y recargar servicio SSH.
3. Confirmar estado efectivo con `sshd -T`.
4. Persistir politica cloud-init con `ssh_pwauth: false`.
5. Ajustar UFW para eliminar `OpenSSH Anywhere` y permitir solo IP operativa.

## Result

- `passwordauthentication no`
- `permitrootlogin no`
- `pubkeyauthentication yes`
- `kbdinteractiveauthentication no`

Riesgo de fuerza bruta SSH reducido de forma material.
