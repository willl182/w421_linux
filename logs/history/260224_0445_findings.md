# Technical Finding / Milestone

**Context**: Cierre de hardening SSH con fail2ban ajustado

## The Discovery

Despues de habilitar fail2ban, el estado correcto del servicio y jail se confirma solo con validacion runtime:

- `systemctl is-active fail2ban` -> `active`
- `fail2ban-client ping` -> `Server replied: pong`
- `fail2ban-client status` -> `Jail list: sshd`
- `fail2ban-client status sshd` -> filtro y acciones activos

## The Solution

1. Instalar y habilitar fail2ban.
2. Validar config con `fail2ban-client -t`.
3. Reiniciar y verificar status/ping/jail.
4. Ajustar jail SSH para este host (`backend=systemd`, `maxretry=5`, `findtime=10m`, `bantime=1h`).
5. Definir `ignoreip` para localhost, tailnet y IP operativa.

## Result

Hardening base de acceso remoto cerrado:

- SSH solo por llave + UFW por IP.
- fail2ban activo protegiendo intentos repetidos sobre `sshd`.
