# Problem Record (Troubleshooting)

**Severity**: Low

## The Problem

Tras un ajuste inicial de jail, aparecio el error:

- `Failed to access socket path: /var/run/fail2ban/fail2ban.sock`

El sintoma se dio cuando el cliente consulto estado mientras el servicio no habia levantado estable despues de restart.

## The Solution / Workaround

1. Validar configuracion con `fail2ban-client -t`.
2. Reiniciar servicio (`systemctl restart fail2ban`).
3. Confirmar salud con `is-active`, `ping` y `status sshd`.

## Learnings to Preserve

- No confiar en un unico comando de estado; usar secuencia `is-active` + `ping` + `status sshd`.
- El warning `allowipv6 not defined` puede aparecer con valor por defecto `auto` y no bloquea el arranque.
