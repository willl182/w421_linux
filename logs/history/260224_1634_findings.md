# Technical Finding / Milestone

**Context**: OpenClaw Gmail realtime + Telegram en VPS Docker (headless)

## The Discovery

- El password de keyring entregado por el usuario (`GOG_KEYRING_PASSWORD`) desbloquea correctamente el token OAuth de `gog`.
- Con ese env presente, `gog gmail watch serve` arranca sin el error `no TTY available for keyring file backend password prompt`.
- El pipeline tecnico local vuelve a funcionar en modo manual: `gmail-pubsub -> /hooks/gmail -> embedded run`.
- El bloqueo de announces por gateway (`pairing required`) quedo corregido previamente al reparar pairing/scopes y permisos de identidad.
