# Problem Record (Troubleshooting)

**Severity**: High

## The Problem

Tras persistir envs y reiniciar por compose, OpenClaw falla al iniciar con:

`Invalid config: hooks.token must not match gateway auth token`

## User-Requested Corrections

- Guardar la sesion con estado real de avance, correcciones aplicadas y bloqueo actual.

## The Solution / Workaround

- Se verifico que `/hostinger/server.mjs` reescribe `openclaw.json` y fuerza `hooks.token` igual a `OPENCLAW_GATEWAY_TOKEN`.
- Se aplico workaround manual editando `/data/.openclaw/openclaw.json` para separar tokens y arrancar `openclaw gateway run` manualmente con env correcto.
- Este workaround no persiste si vuelve a ejecutarse el bootstrap del wrapper.

## Learnings to Preserve

- En esta imagen Hostinger, `server.mjs` actua como bootstrap opinionado y puede invalidar configuracion avanzada de hooks.
- Pairing/keyring no son ya el problema raiz actual; el problema raiz es configuracion regenerada en arranque.
- Para estabilidad real, hay que modificar el bootstrap (o su fuente de env/config) para mantener `hooks.token` distinto de `gateway.auth.token`.
