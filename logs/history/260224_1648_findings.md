# Technical Finding / Milestone

**Context**: Persistencia post-reboot para OpenClaw Gmail webhook en VPS Hostinger

## The Discovery

- Se monto automatizacion post-reboot en host con `systemd` para aplicar fix de config en contenedor OpenClaw.
- El servicio `openclaw-postboot-fix.service` ejecuta `/usr/local/bin/openclaw-postboot-fix.sh`.
- El fix deja `hooks.token` distinto de `gateway.auth.token`, eliminando el fallo de validacion que tumbaba gateway al arranque.
- `GOG_KEYRING_PASSWORD` ya persistido en `.env` mantiene `gog` watcher sin errores de keyring no-TTY.
- Verificacion de funcionamiento: contenedor `Up`, gateway escuchando en `ws://127.0.0.1:18789`, watcher Gmail activo y push sintetico HTTP 200.

## Key Files Affected

- `/usr/local/bin/openclaw-postboot-fix.sh`
- `/etc/systemd/system/openclaw-postboot-fix.service`
- `/docker/openclaw-hacw/.env`
