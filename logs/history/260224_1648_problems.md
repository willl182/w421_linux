# Problem Record (Troubleshooting)

**Severity**: High

## The Problem

- Primer intento de parchear `server.mjs` genero sintaxis invalida (`\"` incrustado) y produjo loop de reinicio del contenedor.

## User-Requested Corrections

- Montar fix automatico post-reboot en el VPS y documentarlo.

## The Solution / Workaround

- Se desactivo temporalmente restart policy para romper el loop.
- Se recupero y corrigio `server.mjs` del contenedor para usar `OPENCLAW_HOOKS_TOKEN` sin escapar mal comillas.
- Se reinstalo script postboot corregido y se habilito servicio systemd.
- Se restauro arranque normal de contenedor y se valido estabilidad.

## Learnings to Preserve

- En parches JS minificados via `node -e`, un `\\\"` de mas puede introducir backslashes literales y romper parser.
- Para fixes persistentes en imagenes gestionadas, conviene automatizar correccion host-side con systemd en lugar de cambios manuales fragiles.
