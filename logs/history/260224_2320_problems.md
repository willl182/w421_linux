# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

- Configuraciones pegadas en UI fallaban con `JSON5: invalid character` por usar sintaxis tipo objeto JS (strings sin comillas).
- Intento de setear `agents.defaults.models.zai/glm-4.7.alias` por path fallo por el punto en `4.7`.
- Se intento usar `openclaw model list`, pero ese subcomando no existe.

## User-Requested Corrections

- Documentar que funciono y que no funciono, omitiendo API key.
- Crear una pagina operativa para repetir la configuracion sin errores.

## The Solution / Workaround

- Usar comandos `openclaw config set` para `primary`, `fallbacks` y `agents.list[0].model`.
- Para alias con version `4.7`, setear el objeto completo de `agents.defaults.models` con JSON en una sola operacion.
- Usar `openclaw models list` para validar disponibilidad de modelos.

## Learnings to Preserve

- Si el parser marca errores en caracteres sueltos (`o`, `.`, `/`), casi siempre hay strings sin comillas.
- En paths de configuracion, claves con punto se interpretan como separadores; para esos casos conviene set por objeto completo.
- Cuando la ruta parece vacia en host, revisar mounts o editar dentro del contenedor.
